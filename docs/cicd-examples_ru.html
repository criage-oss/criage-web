Примеры интеграции CI/CD

Полное руководство по интеграции пакетного менеджера Criage в ваши CI/CD конвейеры.

Обзор

Criage может быть легко интегрирован в различные CI/CD платформы для автоматизации управления пакетами, установки зависимостей и публикации пакетов. Это руководство предоставляет практические примеры для популярных платформ.

🦊GitLab CI

Полные примеры конвейеров GitLab CI с интеграцией Docker и многоэтапными сборками.

🏗️Jenkins

Скрипты конвейеров Jenkins и конфигурации заданий для автоматизированного управления пакетами.

🐙GitHub Actions

Рабочие процессы GitHub Actions для сборки, тестирования и публикации пакетов.

Предварительные требования: Эти примеры предполагают, что у вас настроен и доступен из CI/CD среды сервер репозитория Criage. См. документацию сервера репозитория для инструкций по установке.

GitLab CI

Примеры интеграции GitLab CI для различных сценариев использования.

Базовая установка пакетов

Простой конвейер, который устанавливает зависимости с помощью Criage:

.gitlab-ci.yml
stages:
  - dependencies
  - build
  - test

variables:
  CRIAGE_REPO_URL: "https://packages.example.com"

install_dependencies:
  stage: dependencies
  image: golang:1.21
  before_script:
    - apt-get update && apt-get install -y wget
    - wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
    - chmod +x criage && mv criage /usr/local/bin/
  script:
    - criage repo add main $CRIAGE_REPO_URL
    - criage install --from-config
  artifacts:
    paths:
      - ./packages/
    expire_in: 1 hour

build_project:
  stage: build
  image: golang:1.21
  dependencies:
    - install_dependencies
  script:
    - go build -o app .
  artifacts:
    paths:
      - app

Конвейер публикации пакетов

Продвинутый конвейер, который собирает и публикует пакеты в ваш репозиторий:

.gitlab-ci.yml
stages:
  - build
  - package
  - publish

variables:
  CRIAGE_REPO_URL: "https://packages.example.com"
  PACKAGE_NAME: "my-awesome-package"

# Установка Criage в шаблоне before_script
.install_criage: &install_criage
  before_script:
    - apt-get update && apt-get install -y wget
    - wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
    - chmod +x criage && mv criage /usr/local/bin/

build_package:
  stage: build
  image: golang:1.21
  <<: *install_criage
  script:
    - go build -o $PACKAGE_NAME .
    - mkdir -p package/bin
    - cp $PACKAGE_NAME package/bin/
    - cp README.md package/
  artifacts:
    paths:
      - package/

create_package:
  stage: package
  image: golang:1.21
  <<: *install_criage
  dependencies:
    - build_package
  script:
    - cd package
    - criage init --name $PACKAGE_NAME --version $CI_COMMIT_TAG
    - criage build
  artifacts:
    paths:
      - package/*.criage
  only:
    - tags

publish_package:
  stage: publish
  image: golang:1.21
  <<: *install_criage
  dependencies:
    - create_package
  script:
    - criage repo add main $CRIAGE_REPO_URL
    - criage publish package/*.criage --token $CRIAGE_UPLOAD_TOKEN
  only:
    - tags
  when: manual

Мультиплатформенная сборка

Сборка пакетов для нескольких архитектур:

.gitlab-ci.yml
build_linux_amd64:
  stage: build
  image: golang:1.21
  <<: *install_criage
  variables:
    GOOS: linux
    GOARCH: amd64
  script:
    - go build -o ${PACKAGE_NAME}-${GOOS}-${GOARCH} .
    - mkdir -p packages/${GOOS}-${GOARCH}/bin
    - cp ${PACKAGE_NAME}-${GOOS}-${GOARCH} packages/${GOOS}-${GOARCH}/bin/${PACKAGE_NAME}
    - cd packages/${GOOS}-${GOARCH}
    - criage init --name $PACKAGE_NAME --version $CI_COMMIT_TAG --arch ${GOARCH}
    - criage build
  artifacts:
    paths:
      - packages/${GOOS}-${GOARCH}/*.criage

build_linux_arm64:
  extends: build_linux_amd64
  variables:
    GOOS: linux
    GOARCH: arm64
Совет профессионала: Используйте секретные переменные GitLab для безопасного хранения вашего CRIAGE_UPLOAD_TOKEN. Перейдите в Настройки проекта > CI/CD > Переменные.

Jenkins

Примеры интеграции Jenkins с использованием декларативных и скриптовых конвейеров.

Декларативный конвейер

Современный конвейер Jenkins с использованием декларативного синтаксиса:

Jenkinsfile
pipeline {
    agent {
        docker {
            image 'golang:1.21'
            args '-v /var/run/docker.sock:/var/run/docker.sock'
        }
    }

    environment {
        CRIAGE_REPO_URL = 'https://packages.example.com'
        CRIAGE_UPLOAD_TOKEN = credentials('criage-upload-token')
        PACKAGE_NAME = 'my-package'
    }

    stages {
        stage('Настройка Criage') {
            steps {
                sh '''
                    apt-get update && apt-get install -y wget
                    wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
                    chmod +x criage && mv criage /usr/local/bin/
                    criage --version
                '''
            }
        }

        stage('Установка зависимостей') {
            steps {
                sh '''
                    criage repo add main $CRIAGE_REPO_URL
                    criage install --from-config
                '''
            }
        }

        stage('Сборка') {
            steps {
                sh '''
                    go build -o $PACKAGE_NAME .
                    mkdir -p package/bin
                    cp $PACKAGE_NAME package/bin/
                    cp README.md package/
                '''
            }
        }

        stage('Тестирование') {
            steps {
                sh 'go test ./...'
            }
        }

        stage('Создание пакета') {
            when {
                buildingTag()
            }
            steps {
                sh '''
                    cd package
                    criage init --name $PACKAGE_NAME --version $TAG_NAME
                    criage build
                '''
                archiveArtifacts artifacts: 'package/*.criage', fingerprint: true
            }
        }

        stage('Публикация') {
            when {
                allOf {
                    buildingTag()
                    branch 'main'
                }
            }
            steps {
                input message: 'Опубликовать пакет в репозиторий?', ok: 'Опубликовать'
                sh '''
                    criage publish package/*.criage --token $CRIAGE_UPLOAD_TOKEN
                '''
            }
        }
    }

    post {
        always {
            cleanWs()
        }
        success {
            echo 'Конвейер выполнен успешно!'
        }
        failure {
            echo 'Конвейер завершился с ошибкой!'
        }
    }
}

Скриптовый конвейер

Традиционный скриптовый конвейер Jenkins:

Jenkinsfile
node {
    def packageName = 'my-package'
    def criageRepoUrl = 'https://packages.example.com'
    
    stage('Получение кода') {
        checkout scm
    }
    
    stage('Настройка') {
        sh '''
            apt-get update && apt-get install -y wget
            wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
            chmod +x criage && mv criage /usr/local/bin/
        '''
    }
    
    stage('Зависимости') {
        sh """
            criage repo add main ${criageRepoUrl}
            criage install --from-config
        """
    }
    
    stage('Сборка') {
        sh """
            go build -o ${packageName} .
            mkdir -p package/bin
            cp ${packageName} package/bin/
        """
    }
    
    stage('Тестирование') {
        sh 'go test ./...'
    }
    
    if (env.TAG_NAME) {
        stage('Создание пакета') {
            sh """
                cd package
                criage init --name ${packageName} --version ${env.TAG_NAME}
                criage build
            """
            archiveArtifacts artifacts: 'package/*.criage'
        }
        
        stage('Публикация') {
            input message: 'Опубликовать в репозиторий?'
            withCredentials([string(credentialsId: 'criage-upload-token', variable: 'TOKEN')]) {
                sh '''
                    criage publish package/*.criage --token $TOKEN
                '''
            }
        }
    }
}

Конфигурация задания Freestyle

Для команд, предпочитающих GUI конфигурацию, вот shell скрипт для задания freestyle:

🔧 Скрипт сборки Freestyle задания

#!/bin/bash
set -e

# Конфигурация
CRIAGE_REPO_URL="https://packages.example.com"
PACKAGE_NAME="my-package"

# Установка Criage
if ! command -v criage &> /dev/null; then
    echo "Установка Criage..."
    wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
    chmod +x criage
    sudo mv criage /usr/local/bin/
fi

# Настройка репозитория
criage repo add main $CRIAGE_REPO_URL

# Установка зависимостей
if [ -f "criage.yaml" ]; then
    echo "Установка зависимостей..."
    criage install --from-config
fi

# Сборка проекта
echo "Сборка проекта..."
go build -o $PACKAGE_NAME .

# Запуск тестов
echo "Запуск тестов..."
go test ./...

# Создание пакета если существует тег
if [ ! -z "$GIT_TAG_NAME" ]; then
    echo "Создание пакета..."
    mkdir -p package/bin
    cp $PACKAGE_NAME package/bin/
    cp README.md package/
    
    cd package
    criage init --name $PACKAGE_NAME --version $GIT_TAG_NAME
    criage build
    
    # Публикация если доступен токен загрузки
    if [ ! -z "$CRIAGE_UPLOAD_TOKEN" ]; then
        echo "Публикация пакета..."
        criage publish *.criage --token $CRIAGE_UPLOAD_TOKEN
    fi
fi

echo "Сборка завершена успешно!"
Замечание по безопасности: Всегда храните чувствительные токены, такие как CRIAGE_UPLOAD_TOKEN, как учётные данные Jenkins, а не как обычный текст в скриптах.

GitHub Actions

Рабочие процессы GitHub Actions для интеграции Criage.

Базовый рабочий процесс

Простой рабочий процесс для сборки и тестирования с Criage:

.github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

env:
  CRIAGE_REPO_URL: https://packages.example.com
  PACKAGE_NAME: my-package

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Настройка Go
      uses: actions/setup-go@v4
      with:
        go-version: '1.21'
    
    - name: Установка Criage
      run: |
        wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
        chmod +x criage
        sudo mv criage /usr/local/bin/
    
    - name: Настройка репозитория Criage
      run: |
        criage repo add main ${{ env.CRIAGE_REPO_URL }}
    
    - name: Установка зависимостей
      run: |
        criage install --from-config
    
    - name: Сборка
      run: |
        go build -o ${{ env.PACKAGE_NAME }} .
    
    - name: Тестирование
      run: |
        go test ./...
    
    - name: Загрузка артефакта сборки
      uses: actions/upload-artifact@v3
      with:
        name: ${{ env.PACKAGE_NAME }}
        path: ${{ env.PACKAGE_NAME }}

Рабочий процесс релиза

Автоматизированная публикация пакетов при релизах по тегам:

.github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

env:
  CRIAGE_REPO_URL: https://packages.example.com
  PACKAGE_NAME: my-package

jobs:
  release:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Настройка Go
      uses: actions/setup-go@v4
      with:
        go-version: '1.21'
    
    - name: Установка Criage
      run: |
        wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
        chmod +x criage
        sudo mv criage /usr/local/bin/
    
    - name: Получение версии
      id: version
      run: echo "VERSION=${GITHUB_REF#refs/tags/}" >> $GITHUB_OUTPUT
    
    - name: Сборка
      run: |
        go build -o ${{ env.PACKAGE_NAME }} .
    
    - name: Тестирование
      run: |
        go test ./...
    
    - name: Создание пакета
      run: |
        mkdir -p package/bin
        cp ${{ env.PACKAGE_NAME }} package/bin/
        cp README.md LICENSE package/
        cd package
        criage init --name ${{ env.PACKAGE_NAME }} --version ${{ steps.version.outputs.VERSION }}
        criage build
    
    - name: Публикация пакета
      env:
        CRIAGE_UPLOAD_TOKEN: ${{ secrets.CRIAGE_UPLOAD_TOKEN }}
      run: |
        criage repo add main ${{ env.CRIAGE_REPO_URL }}
        criage publish package/*.criage --token $CRIAGE_UPLOAD_TOKEN
    
    - name: Загрузка пакета в релиз
      uses: actions/upload-release-asset@v1
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
      with:
        upload_url: ${{ github.event.release.upload_url }}
        asset_path: package/${{ env.PACKAGE_NAME }}-${{ steps.version.outputs.VERSION }}.criage
        asset_name: ${{ env.PACKAGE_NAME }}-${{ steps.version.outputs.VERSION }}.criage
        asset_content_type: application/octet-stream

Матричная сборка

Сборка пакетов для нескольких платформ:

.github/workflows/multi-platform.yml
name: Multi-Platform Build

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        goos: [linux, windows, darwin]
        goarch: [amd64, arm64]
        exclude:
          - goos: windows
            goarch: arm64
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Настройка Go
      uses: actions/setup-go@v4
      with:
        go-version: '1.21'
    
    - name: Установка Criage
      run: |
        wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
        chmod +x criage
        sudo mv criage /usr/local/bin/
    
    - name: Сборка
      env:
        GOOS: ${{ matrix.goos }}
        GOARCH: ${{ matrix.goarch }}
      run: |
        VERSION=${GITHUB_REF#refs/tags/}
        BINARY_NAME=my-package
        if [ "$GOOS" = "windows" ]; then
          BINARY_NAME="${BINARY_NAME}.exe"
        fi
        
        go build -o $BINARY_NAME .
        
        mkdir -p package/bin
        cp $BINARY_NAME package/bin/
        cp README.md package/
        
        cd package
        criage init --name my-package --version $VERSION --os $GOOS --arch $GOARCH
        criage build
    
    - name: Загрузка артефакта пакета
      uses: actions/upload-artifact@v3
      with:
        name: packages-${{ matrix.goos }}-${{ matrix.goarch }}
        path: package/*.criage

Лучшие практики

Безопасность

  • Управление токенами: Всегда храните токены загрузки как секреты/учётные данные
  • Сетевая безопасность: Используйте HTTPS для URL репозиториев
  • Контроль доступа: Ограничьте доступ CI/CD runner'ов к серверам репозиториев
  • Подпись артефактов: Рассмотрите подпись пакетов для проверки целостности

Производительность

  • Кэширование: Кэшируйте бинарный файл Criage и установленные пакеты между сборками
  • Параллельные сборки: Используйте матричные сборки для мультиплатформенных пакетов
  • Оптимизация зависимостей: Закрепляйте конкретные версии пакетов в продакшене
  • Этапы сборки: Отделяйте установку зависимостей от сборки

Надёжность

  • Обработка ошибок: Всегда используйте set -e в shell скриптах
  • Логика повтора: Реализуйте повтор для сетевых операций
  • Проверки работоспособности: Проверяйте подключение к репозиторию перед операциями
  • Стратегии отката: Имейте резервные источники репозиториев
Совет: Используйте шаблоны конвейеров или общие библиотеки для поддержания согласованности между проектами.

Решение проблем

Распространённые проблемы

Бинарный файл Criage не найден

# Проверка установки
which criage
criage --version

# Переустановка при необходимости
wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
chmod +x criage
sudo mv criage /usr/local/bin/

Проблемы подключения к репозиторию

# Тест подключения к репозиторию
curl -I https://packages.example.com/api/v1/

# Отладка конфигурации Criage
criage repo list
criage config show

# Сброс конфигурации репозитория
criage repo remove main
criage repo add main https://packages.example.com

Проблемы с токеном загрузки

# Тест валидности токена
curl -H "Authorization: Bearer $TOKEN" \
     https://packages.example.com/api/v1/stats

# Проверка формата токена (должен быть буквенно-цифровым)
echo $TOKEN | wc -c  # Должно быть 32+ символов

Режим отладки

Включение подробного логирования для диагностики:

# Включение режима отладки
export CRIAGE_DEBUG=true
criage install --verbose

# Проверка логов
criage logs --last 50
Получение помощи: Если вы столкнулись с проблемами, не освещёнными здесь, проверьте GitHub Issues или создайте новую проблему с деталями вашей CI/CD платформы и логами ошибок.
