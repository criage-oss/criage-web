CI/CD Integration Examples

Comprehensive guide for integrating Criage package manager into your CI/CD pipelines.

Overview

Criage can be seamlessly integrated into various CI/CD platforms to automate package management, dependency installation, and package publishing. This guide provides practical examples for popular platforms.

Prerequisites: These examples assume you have a Criage repository server set up and accessible from your CI/CD environment. See the Repository Server documentation for setup instructions.

GitLab CI

GitLab CI integration examples for various use cases.

Basic Package Installation

Simple pipeline that installs dependencies using Criage:

.gitlab-ci.yml
stages:
  - dependencies
  - build
  - test

variables:
  CRIAGE_REPO_URL: "https://packages.example.com"

install_dependencies:
  stage: dependencies
  image: golang:1.21
  before_script:
    - apt-get update && apt-get install -y wget
    - wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
    - chmod +x criage && mv criage /usr/local/bin/
  script:
    - criage repo add main $CRIAGE_REPO_URL
    - criage install --from-config
  artifacts:
    paths:
      - ./packages/
    expire_in: 1 hour

build_project:
  stage: build
  image: golang:1.21
  dependencies:
    - install_dependencies
  script:
    - go build -o app .
  artifacts:
    paths:
      - app

Package Publishing Pipeline

Advanced pipeline that builds and publishes packages to your repository:

.gitlab-ci.yml
stages:
  - build
  - package
  - publish

variables:
  CRIAGE_REPO_URL: "https://packages.example.com"
  PACKAGE_NAME: "my-awesome-package"

# Install Criage in before_script template
.install_criage: &install_criage
  before_script:
    - apt-get update && apt-get install -y wget
    - wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
    - chmod +x criage && mv criage /usr/local/bin/

build_package:
  stage: build
  image: golang:1.21
  <<: *install_criage
  script:
    - go build -o $PACKAGE_NAME .
    - mkdir -p package/bin
    - cp $PACKAGE_NAME package/bin/
    - cp README.md package/
  artifacts:
    paths:
      - package/

create_package:
  stage: package
  image: golang:1.21
  <<: *install_criage
  dependencies:
    - build_package
  script:
    - cd package
    - criage init --name $PACKAGE_NAME --version $CI_COMMIT_TAG
    - criage build
  artifacts:
    paths:
      - package/*.criage
  only:
    - tags

publish_package:
  stage: publish
  image: golang:1.21
  <<: *install_criage
  dependencies:
    - create_package
  script:
    - criage repo add main $CRIAGE_REPO_URL
    - criage publish package/*.criage --token $CRIAGE_UPLOAD_TOKEN
  only:
    - tags
  when: manual

Multi-Platform Build

Build packages for multiple architectures:

.gitlab-ci.yml
build_linux_amd64:
  stage: build
  image: golang:1.21
  <<: *install_criage
  variables:
    GOOS: linux
    GOARCH: amd64
  script:
    - go build -o ${PACKAGE_NAME}-${GOOS}-${GOARCH} .
    - mkdir -p packages/${GOOS}-${GOARCH}/bin
    - cp ${PACKAGE_NAME}-${GOOS}-${GOARCH} packages/${GOOS}-${GOARCH}/bin/${PACKAGE_NAME}
    - cd packages/${GOOS}-${GOARCH}
    - criage init --name $PACKAGE_NAME --version $CI_COMMIT_TAG --arch ${GOARCH}
    - criage build
  artifacts:
    paths:
      - packages/${GOOS}-${GOARCH}/*.criage

build_linux_arm64:
  extends: build_linux_amd64
  variables:
    GOOS: linux
    GOARCH: arm64
Pro Tip: Use GitLab's secret variables to store your CRIAGE_UPLOAD_TOKEN securely. Go to Project Settings > CI/CD > Variables.

Jenkins

Jenkins integration examples using both Declarative and Scripted pipelines.

Declarative Pipeline

Modern Jenkins pipeline using declarative syntax:

Jenkinsfile
pipeline {
    agent {
        docker {
            image 'golang:1.21'
            args '-v /var/run/docker.sock:/var/run/docker.sock'
        }
    }

    environment {
        CRIAGE_REPO_URL = 'https://packages.example.com'
        CRIAGE_UPLOAD_TOKEN = credentials('criage-upload-token')
        PACKAGE_NAME = 'my-package'
    }

    stages {
        stage('Setup Criage') {
            steps {
                sh '''
                    apt-get update && apt-get install -y wget
                    wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
                    chmod +x criage && mv criage /usr/local/bin/
                    criage --version
                '''
            }
        }

        stage('Install Dependencies') {
            steps {
                sh '''
                    criage repo add main $CRIAGE_REPO_URL
                    criage install --from-config
                '''
            }
        }

        stage('Build') {
            steps {
                sh '''
                    go build -o $PACKAGE_NAME .
                    mkdir -p package/bin
                    cp $PACKAGE_NAME package/bin/
                    cp README.md package/
                '''
            }
        }

        stage('Test') {
            steps {
                sh 'go test ./...'
            }
        }

        stage('Package') {
            when {
                buildingTag()
            }
            steps {
                sh '''
                    cd package
                    criage init --name $PACKAGE_NAME --version $TAG_NAME
                    criage build
                '''
                archiveArtifacts artifacts: 'package/*.criage', fingerprint: true
            }
        }

        stage('Publish') {
            when {
                allOf {
                    buildingTag()
                    branch 'main'
                }
            }
            steps {
                input message: 'Publish package to repository?', ok: 'Publish'
                sh '''
                    criage publish package/*.criage --token $CRIAGE_UPLOAD_TOKEN
                '''
            }
        }
    }

    post {
        always {
            cleanWs()
        }
        success {
            echo 'Pipeline completed successfully!'
        }
        failure {
            echo 'Pipeline failed!'
        }
    }
}

Scripted Pipeline

Traditional Jenkins scripted pipeline:

Jenkinsfile
node {
    def packageName = 'my-package'
    def criageRepoUrl = 'https://packages.example.com'
    
    stage('Checkout') {
        checkout scm
    }
    
    stage('Setup') {
        sh '''
            apt-get update && apt-get install -y wget
            wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
            chmod +x criage && mv criage /usr/local/bin/
        '''
    }
    
    stage('Dependencies') {
        sh """
            criage repo add main ${criageRepoUrl}
            criage install --from-config
        """
    }
    
    stage('Build') {
        sh """
            go build -o ${packageName} .
            mkdir -p package/bin
            cp ${packageName} package/bin/
        """
    }
    
    stage('Test') {
        sh 'go test ./...'
    }
    
    if (env.TAG_NAME) {
        stage('Package') {
            sh """
                cd package
                criage init --name ${packageName} --version ${env.TAG_NAME}
                criage build
            """
            archiveArtifacts artifacts: 'package/*.criage'
        }
        
        stage('Publish') {
            input message: 'Publish to repository?'
            withCredentials([string(credentialsId: 'criage-upload-token', variable: 'TOKEN')]) {
                sh '''
                    criage publish package/*.criage --token $TOKEN
                '''
            }
        }
    }
}

Freestyle Job Configuration

For teams preferring GUI configuration, here's a shell script for a freestyle job:

🔧 Freestyle Job Build Script

#!/bin/bash
set -e

# Configuration
CRIAGE_REPO_URL="https://packages.example.com"
PACKAGE_NAME="my-package"

# Install Criage
if ! command -v criage &> /dev/null; then
    echo "Installing Criage..."
    wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
    chmod +x criage
    sudo mv criage /usr/local/bin/
fi

# Setup repository
criage repo add main $CRIAGE_REPO_URL

# Install dependencies
if [ -f "criage.yaml" ]; then
    echo "Installing dependencies..."
    criage install --from-config
fi

# Build project
echo "Building project..."
go build -o $PACKAGE_NAME .

# Run tests
echo "Running tests..."
go test ./...

# Package if tag exists
if [ ! -z "$GIT_TAG_NAME" ]; then
    echo "Creating package..."
    mkdir -p package/bin
    cp $PACKAGE_NAME package/bin/
    cp README.md package/
    
    cd package
    criage init --name $PACKAGE_NAME --version $GIT_TAG_NAME
    criage build
    
    # Publish if upload token is available
    if [ ! -z "$CRIAGE_UPLOAD_TOKEN" ]; then
        echo "Publishing package..."
        criage publish *.criage --token $CRIAGE_UPLOAD_TOKEN
    fi
fi

echo "Build completed successfully!"
Security Note: Always store sensitive tokens like CRIAGE_UPLOAD_TOKEN as Jenkins credentials, not as plain text in scripts.

GitHub Actions

GitHub Actions workflows for Criage integration.

Basic Workflow

Simple workflow for building and testing with Criage:

.github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

env:
  CRIAGE_REPO_URL: https://packages.example.com
  PACKAGE_NAME: my-package

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Go
      uses: actions/setup-go@v4
      with:
        go-version: '1.21'
    
    - name: Install Criage
      run: |
        wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
        chmod +x criage
        sudo mv criage /usr/local/bin/
    
    - name: Setup Criage Repository
      run: |
        criage repo add main ${{ env.CRIAGE_REPO_URL }}
    
    - name: Install Dependencies
      run: |
        criage install --from-config
    
    - name: Build
      run: |
        go build -o ${{ env.PACKAGE_NAME }} .
    
    - name: Test
      run: |
        go test ./...
    
    - name: Upload Build Artifact
      uses: actions/upload-artifact@v3
      with:
        name: ${{ env.PACKAGE_NAME }}
        path: ${{ env.PACKAGE_NAME }}

Release Workflow

Automated package publishing on tag releases:

.github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

env:
  CRIAGE_REPO_URL: https://packages.example.com
  PACKAGE_NAME: my-package

jobs:
  release:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Go
      uses: actions/setup-go@v4
      with:
        go-version: '1.21'
    
    - name: Install Criage
      run: |
        wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
        chmod +x criage
        sudo mv criage /usr/local/bin/
    
    - name: Get version
      id: version
      run: echo "VERSION=${GITHUB_REF#refs/tags/}" >> $GITHUB_OUTPUT
    
    - name: Build
      run: |
        go build -o ${{ env.PACKAGE_NAME }} .
    
    - name: Test
      run: |
        go test ./...
    
    - name: Create Package
      run: |
        mkdir -p package/bin
        cp ${{ env.PACKAGE_NAME }} package/bin/
        cp README.md LICENSE package/
        cd package
        criage init --name ${{ env.PACKAGE_NAME }} --version ${{ steps.version.outputs.VERSION }}
        criage build
    
    - name: Publish Package
      env:
        CRIAGE_UPLOAD_TOKEN: ${{ secrets.CRIAGE_UPLOAD_TOKEN }}
      run: |
        criage repo add main ${{ env.CRIAGE_REPO_URL }}
        criage publish package/*.criage --token $CRIAGE_UPLOAD_TOKEN
    
    - name: Upload Package to Release
      uses: actions/upload-release-asset@v1
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
      with:
        upload_url: ${{ github.event.release.upload_url }}
        asset_path: package/${{ env.PACKAGE_NAME }}-${{ steps.version.outputs.VERSION }}.criage
        asset_name: ${{ env.PACKAGE_NAME }}-${{ steps.version.outputs.VERSION }}.criage
        asset_content_type: application/octet-stream

Matrix Build

Build packages for multiple platforms:

.github/workflows/multi-platform.yml
name: Multi-Platform Build

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        goos: [linux, windows, darwin]
        goarch: [amd64, arm64]
        exclude:
          - goos: windows
            goarch: arm64
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Go
      uses: actions/setup-go@v4
      with:
        go-version: '1.21'
    
    - name: Install Criage
      run: |
        wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
        chmod +x criage
        sudo mv criage /usr/local/bin/
    
    - name: Build
      env:
        GOOS: ${{ matrix.goos }}
        GOARCH: ${{ matrix.goarch }}
      run: |
        VERSION=${GITHUB_REF#refs/tags/}
        BINARY_NAME=my-package
        if [ "$GOOS" = "windows" ]; then
          BINARY_NAME="${BINARY_NAME}.exe"
        fi
        
        go build -o $BINARY_NAME .
        
        mkdir -p package/bin
        cp $BINARY_NAME package/bin/
        cp README.md package/
        
        cd package
        criage init --name my-package --version $VERSION --os $GOOS --arch $GOARCH
        criage build
    
    - name: Upload Package Artifact
      uses: actions/upload-artifact@v3
      with:
        name: packages-${{ matrix.goos }}-${{ matrix.goarch }}
        path: package/*.criage

Best Practices

Security

  • Token Management: Always store upload tokens as secrets/credentials
  • Network Security: Use HTTPS for repository URLs
  • Access Control: Limit CI/CD runner access to repository servers
  • Artifact Signing: Consider signing packages for integrity verification

Performance

  • Caching: Cache Criage binary and installed packages between builds
  • Parallel Builds: Use matrix builds for multi-platform packages
  • Dependency Optimization: Pin specific package versions in production
  • Build Stages: Separate dependency installation from building

Reliability

  • Error Handling: Always use set -e in shell scripts
  • Retry Logic: Implement retry for network operations
  • Health Checks: Verify repository connectivity before operations
  • Fallback Strategies: Have backup repository sources
Tip: Use pipeline templates or shared libraries to maintain consistency across projects.

Troubleshooting

Common Issues

Criage Binary Not Found

# Verify installation
which criage
criage --version

# Reinstall if needed
wget -O criage https://github.com/criage-oss/criage-client/releases/latest/download/criage-linux-amd64
chmod +x criage
sudo mv criage /usr/local/bin/

Repository Connection Issues

# Test repository connectivity
curl -I https://packages.example.com/api/v1/

# Debug Criage configuration
criage repo list
criage config show

# Reset repository configuration
criage repo remove main
criage repo add main https://packages.example.com

Upload Token Issues

# Test token validity
curl -H "Authorization: Bearer $TOKEN" \
     https://packages.example.com/api/v1/stats

# Check token format (should be alphanumeric)
echo $TOKEN | wc -c  # Should be 32+ characters

Debug Mode

Enable verbose logging for troubleshooting:

# Enable debug mode
export CRIAGE_DEBUG=true
criage install --verbose

# Check logs
criage logs --last 50
